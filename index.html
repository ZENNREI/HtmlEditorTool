🛒 Makeshop HTML 変換

    

      なぜ表示が異なるのか？

      Makeshopの独自属性が標準ブラウザでは無視されるため、以下の問題が発生します：

      ① tablewidth="100" → Makeshopでは「幅100%」を意味するが、HTML出力時に style="width: 100px"（100ピクセル）として書き出される。標準ブラウザはinline styleを優先するため100px幅になってしまう。

      ② textalign / verticalalign / borderstyle / bordercolor → Makeshop固有の非標準属性。標準ブラウザでは完全に無視される。

      ③ height / width / align / valign / bgcolor → 非推奨のHTML属性。一部のブラウザ/状況では無視される場合がある。

      ④ 空の<tr>行 → rowspanだけを持つ行（「5月」など）は、列が足りないため空行として見えることがある。
    

    

      変換内容

      ✅ tablewidth="N" → style="width: N%"（px指定を%に上書き）

      ✅ height="N" on td/tr → style="height: Npx"

      ✅ width="N%" on td/th → style="width: N%"

      ✅ align="center" → style="text-align: center"

      ✅ valign="middle" → style="vertical-align: middle"

      ✅ bgcolor="#fff" → style="background-color: #fff"
